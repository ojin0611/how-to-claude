사례 둘 · 자동화

매달 반복하는 보고서를 자동화한 지인의 이야기

제약 관련 업무에서 매달 만드는 품질관리 보고서가 있습니다. 양식도, 검사 항목도, 단위 표기도 정해져 있어요 — 사람이 매번 맞춰 쓰는 게 지루하고 실수가 나기 좋은 일. 이 지인은 Claude를 처음 써보는 상태에서 "이걸 AI한테 넘겨보자"로 시작했습니다. 이 사례는 카드 하나·셋·넷·여섯의 바탕이 된 이야기예요.

단계 1처음엔 그냥 "해줘"로 시작했다

결과가 매번 조금씩 달랐다.

첫 시도는 단순했어요. 과거 보고서 파일을 한두 개 보여주고 "이런 스타일로 이번 달 것도 만들어줘." 결과는 그럭저럭 나왔습니다.

문제는 매번 조금씩 다른 양식이 나온다는 점이었어요. 어떤 날은 표 순서가 바뀌고, 어떤 날은 단위 표기가 달라지고, 빈 칸은 때에 따라 -, N/A, 공란을 왔다갔다 했습니다.

사람이 매번 "이거 아니야, 다시"를 말하고 있었어요. 결국 내가 직접 하는 것과 시간이 비슷해지는 지점이 왔습니다.

단계 2한 장짜리 설명서를 써봤다

"시험 항목은 이 순서, 단위는 이 표기, 빈 값은 이렇게."

한 장짜리 설명서를 쓰고 매 대화의 맨 앞에 붙였습니다. 내용은 길 필요가 없었어요 —

시험 항목 순서 / 표 구성 / 단위 표기 / 빈 값 표기 / 첨부 없으면 "해당 없음"으로 표시 / 최종 승인란은 비워 둘 것

10줄짜리 문서였는데, 그 다음부터 결과가 일관되게 나오기 시작했습니다. "해줘"와 "이 규칙으로 해줘" 사이의 차이가 생각보다 컸어요.

이 단계가 카드 하나(설명서)의 원형

좋은 설명서는 길지 않아요. "내가 매번 같은 지적을 반복하고 있구나" 싶을 때, 그 지적을 한 줄로 적어두는 데서 시작합니다.

단계 3설명서가 너무 길어지기 시작했다

제품 종류가 여럿이고, 각 제품마다 규칙이 달랐다.

보고서 대상 제품이 4종류였는데, 각 제품마다 검사 항목도, 단위도, 레이아웃도 달랐어요. 처음엔 한 장 설명서에 모든 제품의 규칙을 다 몰아 적었더니 —

AI가 제품 A의 규칙을 제품 B에 적용하는 실수를 하기 시작했습니다. 규칙끼리 부딪쳐서 AI가 헤매는 상태.

해결은 단순했습니다. 제품마다 설명서를 따로 파일로 분리하고, 해당 제품 작업을 할 때만 그 파일을 꺼내 쓰도록 했어요. 혼선이 사라졌습니다.

이 단계가 카드 넷(쪼개기)의 원형

"운전 중에 요리 매뉴얼을 읽히지 않는다"는 감각. 지금 일과 무관한 규칙은 오히려 방해입니다.

단계 4그런데 중요한 사건이 있었다

AI가 "검토 완료"라고 했는데, 검토한 적이 없었다.

자동화가 안정되자 검토 단계를 AI에게 붙이기 시작했어요. "결과를 네가 한 번 확인해보고 이상 없으면 '완료'라고 말해줘."

AI는 매번 "검토 완료, 이상 없음"이라고 보고했습니다. 그런데 어느 날 우연히 과거 보고서와 결과를 직접 비교해봤더니 —

AI는 정답으로 삼아야 할 과거 보고서 파일과 한 번도 비교한 적이 없었다. 단지 자기가 만든 결과를 자기가 훑어보고 "문제 없어 보인다"고 말한 것뿐이었다.

시험지를 본인이 채점한 셈. "검토 완료"라는 말이 "실제로 대조해봤다"를 뜻하지 않았던 것입니다. 이게 카드 여섯에서 이야기하는 조용한 실패의 가장 전형적인 형태예요.

조용한 실패

가장 무서운 건 AI가 에러를 낼 때가 아닙니다. AI가 자신 있게 틀렸을 때, 우리는 속기 쉬워요. 결과의 모양이 그럴듯하면 검증을 건너뜁니다.

단계 5자기 채점을 버리고 자동 대조를 붙였다

"AI가 결과를 낼 때마다, 사람 손 없이도 과거 보고서와 자동 비교."

✗ 이전 구조

AI가 결과 생성 → AI가 "검토"했다고 보고 → 사람이 믿고 사용 (실은 대조 안 됨)

✓ 지금 구조

AI가 결과 생성 → 자동으로 과거 보고서와 항목 대조 → 차이만 사람에게 보여줌

중요한 건 대조 작업을 AI 본인에게 다시 맡기지 않았다는 점입니다. "네가 했다고 말하는 것"과 "진짜로 된 것"을 분리하기 위해, 대조는 사람이 한 번 정의한 뒤 자동으로 돌아가도록 배선했어요.

이제 "완료"라는 말은 "자동 대조를 통과했다"를 의미합니다. 단어의 뜻이 달라진 거예요.

이 단계가 카드 셋(자동 확인)의 원형

"사람이 매번 점검하지 않아도 되는 구조"는 신뢰의 문제가 아니라 에너지의 문제입니다. 사람이 매번 검사할 체력이 없으면 검사가 결국 빠지게 돼요. 그래서 구조에 박아두는 거예요.

지금 어디까지 왔나

  • 설명서 체계 — 공통 설명서 한 장 + 제품별 설명서 4장
  • 자동 대조 — 생성된 결과를 과거 보고서와 자동 비교해 차이만 리포트
  • 반복 주기 — 매월 1회 실행. 사람은 차이 리포트만 검토하면 됨
