# how-to-claude — 프로젝트 규칙

이 프로젝트는 **비개발자가 Claude를 잘 쓰는 방법**을 공유하기 위한 공개 웹사이트입니다.

- 저장소: `github.com/ojin0611/how-to-claude`
- 배포: GitHub Pages 정적 사이트 (`web/` 디렉터리)
- 독자: 비개발자 (고객, 지인, 방문자)

---

## 🔒 프라이버시 규칙 (엄수)

이 저장소는 **공개**됩니다. 따라서 다음을 반드시 지켜주세요.

### 절대 공개 웹 페이지(`web/**/*.html`)에 넣지 말 것

- **사용자 실명** — 현재 고객 3명(환경: Mac / Samsung / ASUS) 및 과거 사례의 친구·지인 이름 모두 포함
- **식별 가능한 닉네임·핸들** — YouTube 채널명, 인스타그램 핸들, GitHub 사용자명
- **개인 URL** — 개인 채널/레포/SNS 링크
- **구체적 회사명·제품명** — 특정 개인이 소속된 곳을 식별할 수 있는 정보
- **주민번호·전화번호·이메일·주소** — 당연히 제외
- **신원 유추 가능한 조합** — "10년차 스타트업 창업멤버 + 30대 초반 + 홍콩 브이로그" 같이 묶이면 특정 가능한 조합

### 대신 이렇게

- 사용자: "한 친구", "어떤 지인", "고객 A" 등 익명 호칭
- 환경만 언급: "Mac 사용자", "Samsung 노트북 사용자" 등
- 맥락은 일반화: "30대 커리어 전환기", "제약회사 품질관리 담당" 정도의 직무 범주

### 예외 — 메모리(`~/.claude/projects/.../memory/`)

메모리 파일은 공개되지 않습니다. 거기엔 실명·상세 맥락을 기록해도 됩니다.

### 예외 — 공개 인물의 공개 사실

Anthropic·리처드 서튼 같이 공개 활동 인물의 공개 정보(이름·직함·공개된 발언)는 사용 가능합니다.

## 글쓰기 톤·방침

- **대상은 비개발자**: 코드·터미널·영문 약어(AGENTS.md, MCP, hook 등) 지양
- 꼭 써야 하면 먼저 한국어 비유로 설명 (예: MCP → "외부 콘센트")
- 진단 → 카드 → 상세 페이지의 3단 구조를 유지
- 출처가 되는 실제 사례는 개인정보 제거 후 **패턴만** 서술

## 작업 체크리스트 (변경을 반영하기 전에)

- [ ] `web/**/*.html`에 실명이 들어가 있지 않은가?
- [ ] 채널/레포/SNS URL이 외부에 노출되지 않는가?
- [ ] 신원 유추가 가능한 조합이 있진 않은가?
- [ ] 변경사항이 `web/` 밖의 개인 파일(`최종목표*/`, 개인 자료)을 커밋 대상에 포함하지 않는가?

## `.gitignore` 가이드

저장소에서 제외할 것:
- `최종목표*/` — 친구/고객의 개인 작업물
- `dk_PQR-REF-maker-main/` — 타인의 프로젝트 원본
- `intake/`, `_research/`, `cards/` — 비공개 분석 메모
- `.obsidian/`, `.claude/` — 로컬 설정
- 커밋은 원칙적으로 `web/`, `CLAUDE.md`, `README.md`, `.gitignore`만
