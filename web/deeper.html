Claude Code는 무엇으로 이루어져 있나

메인 페이지에서는 Claude Code를 LLM · 에이전트 · 하네스 세 겹으로 설명했어요. 이 페이지는 그 다음 질문에 답합니다 — "그래서 내가 뭘 만들고, 뭘 안 만드는 거야?"

질문 하나

누가 무엇을 만드나

세 겹이 모두 Claude Code에 들어있지만, 만드는 주체가 다릅니다.

층 내용 만드는 사람
① LLM AI의 뇌 — 말을 이해하고 만드는 부분 Anthropic
② 에이전트 뇌에 손발을 붙인 시스템 — 읽기·쓰기·찾기·실행 Claude Code
③ 하네스 에이전트가 일할 방의 배치와 규칙 당신

즉 "Claude Code를 잘 쓴다"는 실질적으로 "③ 하네스를 잘 짠다"는 뜻에 가깝습니다. 이 사이트의 카드 1~6도 전부 ③에 대한 얘기예요.

질문 둘

그럼 사용자가 "새로 만드는 것"은 뭐죠?

새 프로그램을 만든다 = 보통 하네스 한 벌을 짠다는 뜻입니다. 코드가 아니라 자연어로 된 문서와 설정을 쓰는 일이에요.

설명서 한 장 기본

이 프로젝트에서 AI가 지켜야 할 규칙을 한 장에. → 카드 하나

"AI가 ~하면 자동으로 ~가 확인되게" 같은 배선. 사람이 매번 검사하지 않아도 되도록. → 카드 셋

상황별 설명서 쪼개기 기본

규칙이 길어지면 주제·상황별로 파일을 나눠 그때그때 불러오기. → 카드 넷

외부 콘센트 연결 고급

사내 시스템·구글 달력·회사 위키 등을 AI의 손발로 붙이는 것. 기술 이름은 MCP. 당장은 몰라도 되고, 필요해질 때 돌아와서 읽으면 돼요.

질문 셋

에이전트는 뇌와 손발이 전부인가요?

아니요. 에이전트를 더 자세히 열면 이렇게 구성돼 있습니다. 초록 태그는 당신이 설계에 손댈 수 있는 부분, 회색은 Claude Code가 기본 제공하는 부분이에요.

부품 하는 일 예
뇌 기본 생각·판단 Claude 모델
손발 기본 실제 행동 파일 읽기·쓰기·찾기·실행
기억 설계 가능 맥락을 유지하는 능력 대화 중 기억 + 파일로 남기는 장기 기억
반복 고리 기본 생각 → 행동 → 관찰 → 다시 생각 에이전트의 핵심 동작 방식
부하 직원 설계 가능 일을 다른 에이전트에게 맡기기 서브에이전트
숙련 매뉴얼 설계 가능 반복되는 작업 절차를 묶어 재사용 스킬(Skill)
외부 콘센트 설계 가능 바깥 시스템 연결 MCP (사내 DB, 외부 API 등)
기본 Claude Code가 기본으로 제공 — 건드리지 않아도 됨
설계 가능 하네스 짜면서 당신이 정할 수 있는 부분
정리하면 — 에이전트의 몸은 Claude Code가 주고,
그 몸을 어떻게 쓸지는 하네스가 결정합니다.
그리고 그 하네스를 쓰는 사람이, 바로 당신이에요.
